## Spec-Driven Development

Planning artifacts live at `{{PLANNING_ROOT}}/`, configured by `planning-config.json`. Use the installed `sdd-planner` skills for research, specifications, designs, plans, implementation, review, and retrospectives.

Keep artifact status in YAML frontmatter. Update task status and subtask checklists only after verification commands have actually run. Before implementation, read the active plan and phase; after material changes, run the Code Review skill.
