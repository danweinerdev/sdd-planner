## Spec-Driven Development

Planning artifacts live at `{{PLANNING_ROOT}}/`, configured by `planning-config.json`. Use the installed `sdd-planner` skills for research, specifications, designs, plans, implementation, review, debriefs, and validation.

Keep artifact status in YAML frontmatter. Plan and implement each task as one clean, complete, independently bisectable native SCM revision/checkpoint; subtasks are mechanical steps inside that boundary. Review each task's complete diff before finalizing that revision and preserve complete task state in the lifecycle record. Update task status and subtask checklists only after verification commands have actually run and the task's completion-evidence section records exact commands/tools, context, revision/checkpoint, result, and observable evidence. **Git adapter:** commit the verified feature slice before recording evidence and commit lifecycle bookkeeping separately; dirty snapshot folders are fallback-only. Phase completion requires aggregate evidence plus a persisted, frozen, final `Aligned` run of all four `sdd-code-review` lanes. Material changes after review require a fresh full run. Before implementation, read the active plan and phase; after material changes, run the `sdd-code-review` skill.
